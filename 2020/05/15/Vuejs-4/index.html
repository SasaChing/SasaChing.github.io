let & var

比較 var ＆ let

範例一

  • console.log(mom); 在 var 之前能夠產生 undefined 是因為有hoisting特性

hoisting特性：變數和函數的宣告會在編譯階段就被放入記憶體。

  • undefined 已經有預備變數位置
  • let 可避免產生全域變數
  • not defined尚未準備變數空間

作用域

範例二

  • var 作用域是function scope (function內都能取用) 因此使用 var 會產生杰哥
  • let 作用域 block 只在內層大括號內 使用 let 會產生小明 內層並不會影響到外層變數

迴圈

範例三

  • var i=0 是全域變數，執行過程會先走完for迴圈後再把i的總數儲存後，在進行setTimeout，所以並不會依序列出。
  • let i = 0 範圍設在for迴圈內，必然setTimeout能夠跟著迴圈。

const

const 是宣告一個常數，宣告後的變數不能隨意修改。
但是 const 所宣告的如果是物件變數，則物件內的屬性可隨意修改

物件本身是傳參考，只修改參考內的值

陣列方法

需注意瀏覽器的支援（ex: IE8）

forEach( )

將陣列內的每個元素，皆傳入並執行給定的函式一次

JavaScript 也可以使用這類型的陣列方式

map( )

會建立一個新的陣列，其內容為原陣列的每一個元素經由回呼函式運算後所回傳的結果之集合。

filter( )

把所有為 true 值全部回傳，並且回傳為陣列

find ( )

把 true 值只回傳一次，回傳為本身的值，適用於尋找單一條件的值或 id

every ( )

所以結果都為 true ， return 才會是true

some ( )

只要一個結果為 true ， return 就會是true

reduce ( )

用來累積回呼函式回傳值的累加器，累加器是上一次呼叫後，所回傳的累加數值。
