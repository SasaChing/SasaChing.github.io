發表於 2020-05-15 | 分類於 Vue.js筆記 |

基礎元件

Vue Component 簡單來說是能夠把 Html 包裝起來並且使用自定義的標籤，就可以直接運用在網頁上。

  • 元件重複使用，資料都是獨立運作
  • 元件也能包元件
  • 可避免大量資料冗長

傳遞資料

元件內外層傳遞資料方式

  • v-for 的 item 指向 data 裡的陣列資料，傳入 :person="item"，再由外層以 props:['person'] 傳入內層。

x-template

由於再建立 component 時，隨著資料量愈來越多，也會更加難以閱讀，所以使用x-template 能夠在包裝一次template 顯得更加簡潔。

is 屬性

動態切換模板

有些情況例如tbody 內只會出現 tr、td 如果出現了元件或其他tag 則可能會出現跑版、錯誤的產生，所以這時可以使用is來做動態切換模板。

全域＆局部註冊

全域註冊

可讓所有Vue應用程式所使用

局部註冊

只在有註冊component的Vue應用程式所使用

function return

在component 裡使用data必須要加上 function() 並且使用return {}回傳

Props

由外到內的資料傳遞

注意事項

單向數據流

不能直接修改外部props傳進來的內容，不能反向的傳回去。然而要更正這個錯誤，必須在component加上data、function、return 增加一個newUrl來接收修改過的imgUrl，並且到x-template 更正v-model為新的newUrl。

尚未宣告的變數

在進行Ajax行為時，索取資料的時間差，會讓Vue找不到其中某些變數（尚未宣告的變數）而產生的錯誤，而我們可以用v-if來解決狀況。

可以先選擇Ajax所讀進來的資料裡某個項目，例如：phone意味著假設user.phone尚未讀進來時就不要顯示card，來做判斷式，結果能因為時間差而尚未宣告的變數不再產生錯誤。

維持狀態與生命週期

元件若再重新生成使用時，都會完全重新執行一次程式碼且重新一次生命週期，若不需要此行為可以使用keep-alive讓程式碼可以先暫時隱藏，等到啟用時也可以保持原本的狀態。

型別

提早定義props的數據型別，可以避免之後運算時產生型別錯誤。

emit 向外層傳送事件

由內而外傳送資料時必須用事件來觸發

在button-counter元件內新增了@add="addCounter"事件來觸發資料傳送，在元件內template包裝了incrementCounter來執行註冊在methods的add事件，進而觸發addCounter行為傳送。



Slot 插槽替換

當要重複使用元件時，並不是每次都是使用一樣的模板，可能內容需要更改時，就可以使用Slot插槽替換部分內容
